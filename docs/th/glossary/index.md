# อภิธานศัพท์ ISO 20022

อภิธานศัพท์นี้กำหนดคำศัพท์สำคัญ ตัวย่อ และแนวคิดทางเทคนิคที่ใช้ในข้อความ pacs ตามมาตรฐาน ISO 20022 และในเว็บไซต์นี้

## A

**ACH** — Automated Clearing House (สำนักหักบัญชีอัตโนมัติ) เครือข่ายที่ประมวลผลการชำระเงินอิเล็กทรอนิกส์แบบกลุ่มระหว่างสถาบันการเงิน

**AdrLine** — Address Line (บรรทัดที่อยู่) ฟิลด์ที่อยู่ข้อความอิสระในโครงสร้างที่อยู่ไปรษณีย์ ISO 20022 สูงสุด 7 บรรทัด บรรทัดละ 70 ตัวอักษร กำลังถูกแทนที่ด้วยองค์ประกอบที่อยู่แบบมีโครงสร้างสำหรับ CBPR+ ภายในพฤศจิกายน 2026

**ACCP** — Accepted Customer Profile (โปรไฟล์ลูกค้าผ่านการยอมรับ) รหัสสถานะ pacs.002 ที่ระบุว่าการตรวจสอบก่อนหน้า (ไวยากรณ์ โปรไฟล์ลูกค้า) ผ่านแล้ว

**ACSC** — Accepted Settlement Completed (การชำระบัญชีเสร็จสมบูรณ์) รหัสสถานะ pacs.002 ที่ยืนยันว่าการชำระบัญชีในบัญชีผู้ชำระหนี้เสร็จสมบูรณ์แล้ว

**ACSP** — Accepted Settlement in Process (การชำระบัญชีอยู่ระหว่างดำเนินการ) รหัสสถานะ pacs.002 ที่ระบุว่าการตรวจสอบทั้งหมดผ่านและการชำระบัญชีอยู่ระหว่างดำเนินการ

## B

**BAH** — Business Application Header (head.001) ซองมาตรฐานที่ห่อหุ้มข้อความธุรกิจ ISO 20022 สำหรับการขนส่งผ่าน SWIFT ประกอบด้วยข้อมูลการกำหนดเส้นทาง ตัวระบุการกำหนดข้อความ และ BIC ผู้ส่ง/ผู้รับ

**BIC** — Business Identifier Code (ISO 9362) รหัส 8 หรือ 11 ตัวอักษรที่ระบุสถาบันการเงินอย่างไม่ซ้ำกัน รูปแบบ: BBBBCCLL (รหัสธนาคาร + ประเทศ + ที่ตั้ง) พร้อมรหัสสาขา BBB เสริม

## C

**CBPR+** — Cross-Border Payments and Reporting Plus โปรแกรมของ SWIFT สำหรับการย้ายข้อความการชำระเงินข้ามพรมแดนจาก MT เป็น ISO 20022 เริ่มดำเนินการมีนาคม 2023

**CdtTrfTxInf** — Credit Transfer Transaction Information บล็อกการสร้างหลักระดับธุรกรรมใน pacs.008 ประกอบด้วยรายละเอียดการชำระเงิน ฝ่ายต่างๆ จำนวนเงิน และข้อมูลการโอนเงิน

**ChrgBr** — Charge Bearer (ผู้แบกรับค่าธรรมเนียม) ระบุว่าใครจ่ายค่าธรรมเนียมธุรกรรม ค่า: DEBT (ผู้ชำระหนี้), CRED (เจ้าหนี้), SHAR (แบ่ง), SLEV (ระดับบริการ เฉพาะ SEPA)

**CLRG** — การชำระบัญชีผ่านระบบหักบัญชี วิธีการชำระบัญชีที่เงินทุนเคลื่อนผ่านระบบหักบัญชีเช่น TARGET2, EURO1 หรือ CHIPS

**COVE** — การชำระบัญชีวิธีคุ้มครอง วิธีการชำระบัญชีที่การชำระเงินคุ้มครอง pacs.009 แยกต่างหากจัดการเงินทุนระหว่างธนาคารตัวแทน ขณะที่ pacs.008 บรรจุข้อมูลลูกค้าโดยตรง

**CSM** — Clearing and Settlement Mechanism (กลไกหักบัญชีและชำระบัญชี) โครงสร้างพื้นฐานที่ประมวลผลและชำระบัญชีคำสั่งการชำระเงินระหว่างสถาบันที่เข้าร่วม

## D

**Dbtr** — Debtor (ผู้ชำระหนี้) ฝ่ายที่เป็นหนี้เงินทุนและเริ่มการชำระเงิน ใน pacs.008 องค์ประกอบ Dbtr บรรจุชื่อ ที่อยู่ไปรษณีย์ การระบุตัวตน และประเทศที่พำนักของผู้ชำระหนี้

**DbtrAgt** — Debtor Agent (ตัวแทนผู้ชำระหนี้) สถาบันการเงินที่ให้บริการบัญชีของผู้ชำระหนี้และส่งคำสั่ง pacs.008

## E

**E2E ID** — End-to-End Identification (EndToEndId) ข้อมูลอ้างอิงที่กำหนดโดยผู้เริ่มต้นที่ต้องคงอยู่ไม่เปลี่ยนแปลงผ่านตัวแทนทั้งหมดในห่วงโซ่การชำระเงิน ใช้สำหรับการตรวจสอบย้อนกลับระดับลูกค้า

**EPC** — European Payments Council (สภาการชำระเงินแห่งยุโรป) องค์กรที่จัดการกฎเกณฑ์แผนการชำระเงิน SEPA สำหรับการโอนเครดิตและหักบัญชีโดยตรง

## F–X

**FI** — Financial Institution (สถาบันการเงิน) ธนาคารหรือสถาบันอื่นที่เข้าร่วมในการหักบัญชีและชำระบัญชีการชำระเงิน

**FIToFI** — Financial Institution to Financial Institution อธิบายโดเมนระหว่างธนาคารที่ข้อความ pacs ดำเนินการ

**gpi** — Global Payments Innovation ความริเริ่มของ SWIFT เพื่อการชำระเงินข้ามพรมแดนที่เร็วขึ้นและโปร่งใสมากขึ้น ใช้ UETR สำหรับการติดตามจากต้นทางถึงปลายทางผ่าน Tracker บนคลาวด์

**GrpHdr** — Group Header บล็อกข้อมูลเมตาระดับข้อความในข้อความ pacs ประกอบด้วย MsgId, CreDtTm, NbOfTxs, ข้อมูลการชำระบัญชี และข้อมูลประเภทการชำระเงิน

**IBAN** — International Bank Account Number (ISO 13616) รูปแบบหมายเลขบัญชีมาตรฐานที่ใช้สำหรับการชำระเงินข้ามพรมแดนและภายในประเทศ ตรวจสอบโดยใช้ checksum ISO 7064 Mod 97-10

**ISO 20022** — มาตรฐานสากลสำหรับการแลกเปลี่ยนข้อมูลอิเล็กทรอนิกส์ระหว่างสถาบันการเงิน กำหนดโมเดลข้อมูลร่วมและรูปแบบข้อความตาม XML สำหรับการชำระเงิน หลักทรัพย์ การเงินการค้า และโดเมนการเงินอื่นๆ

**LEI** — Legal Entity Identifier (ISO 17442) รหัสตัวอักษรและตัวเลข 20 ตัวที่ระบุนิติบุคคลที่เข้าร่วมในธุรกรรมทางการเงินอย่างไม่ซ้ำกัน ใช้ใน OrgId/LEI สำหรับฝ่ายต่างๆ และ FinInstnId/LEI สำหรับตัวแทน

**MsgId** — Message Identification ตัวระบุเฉพาะสำหรับซองข้อความ กำหนดโดยตัวแทนผู้ส่ง เปลี่ยนในแต่ละ hop ของห่วงโซ่การชำระเงิน

**pacs** — Payments Clearing and Settlement โดเมนธุรกิจ ISO 20022 ที่ครอบคลุมข้อความการชำระเงินระหว่างธนาคาร

**pacs.008** — การโอนเครดิตลูกค้า FI ถึง FI ข้อความระหว่างธนาคารหลักสำหรับการโอนเครดิตลูกค้า แทนที่ MT103

**RTGS** — Real-Time Gross Settlement ระบบการชำระเงินที่ธุรกรรมชำระบัญชีแยกเป็นรายการและแบบเรียลไทม์ (เช่น TARGET2, Fedwire, CHAPS)

**SEPA** — Single Euro Payments Area ความริเริ่มในการบูรณาการการชำระเงินที่ครอบคลุมการโอนเครดิต หักบัญชีโดยตรง และการชำระเงินด้วยบัตรสกุลยูโรใน 36 ประเทศยุโรป

**UETR** — Unique End-to-End Transaction Reference ตัวระบุ UUID v4 ที่สร้างโดยตัวแทนผู้ชำระหนี้และบรรจุไม่เปลี่ยนแปลงตลอดทุกขาของการชำระเงินสำหรับการติดตาม gpi

**XSD** — XML Schema Definition สคีมาที่เป็นทางการกำหนดโครงสร้าง องค์ประกอบ และชนิดข้อมูลของข้อความ XML ISO 20022

**XXE** — XML External Entity ช่องโหว่ด้านความปลอดภัยในการแยกวิเคราะห์ XML pacs008 ป้องกันการโจมตี XXE โดยใช้ defusedxml
