# pacs.003.001.09 — FI to FI Customer Direct Debit

| | |
|---|---|
| **ชื่อ ISO** | FIToFICustomerDirectDebitV09 |
| **สถานะการลงทะเบียน** | Registered |
| **ปี** | 2019 |
| **เวอร์ชัน** | 9 |

## ภาพรวม

ข้อความ pacs.003 ถูกแลกเปลี่ยนระหว่างสถาบันการเงินเพื่อดำเนินการคำสั่งหักบัญชีโดยตรงของลูกค้า ช่วยให้ธนาคารของเจ้าหนี้สามารถเรียกเก็บเงินจากธนาคารของลูกหนี้ในนามของเจ้าหนี้

## องค์ประกอบข้อมูลหลัก

- **GrpHdr** — Group Header พร้อมรหัสระบุข้อความและข้อมูลการชำระบัญชี
- **DrctDbtTxInf** — ข้อมูลธุรกรรมหักบัญชีโดยตรงพร้อมจำนวนเงินและคู่สัญญา
- **Cdtr** — การระบุตัวตนเจ้าหนี้และรายละเอียดบัญชี
- **CdtrAgt** — การระบุตัวตนตัวแทนเจ้าหนี้ (สถาบันเรียกเก็บเงิน)
- **DbtrAgt** — การระบุตัวตนตัวแทนลูกหนี้ (สถาบันชำระเงิน)

## บริบทธุรกิจ

- รองรับโครงการหักบัญชีโดยตรง SEPA Core และ B2B
- ใช้สำหรับการเรียกเก็บเงินแบบเป็นงวด เช่น การสมัครสมาชิก ค่าสาธารณูปโภค และการชำระคืนเงินกู้
- ต้องมีการอ้างอิงคำสั่งมอบอำนาจที่ถูกต้องระหว่างลูกหนี้และเจ้าหนี้
- เปิดใช้งานการเรียกเก็บเงินแบบกลุ่มของคำสั่งหักบัญชีโดยตรงหลายรายการในข้อความเดียว

## บริบท CBPR+ และรูปแบบ

- ข้อกำหนดที่อยู่แบบมีโครงสร้างและการระบุตัวตนคู่สัญญาบังคับใช้กับการหักบัญชีโดยตรงเช่นเดียวกัน
- ข้อมูลที่เกี่ยวข้องกับคำสั่งมอบอำนาจต้องมีโครงสร้างอย่างสมบูรณ์ตั้งแต่เดือนพฤศจิกายน 2026
- แทนที่รูปแบบหักบัญชีโดยตรงแบบ MT104 ดั้งเดิมในขั้นตอนข้ามพรมแดน
- การตรวจสอบการระบุโครงการเจ้าหนี้ถูกบังคับใช้มากขึ้น

## กระแสข้อความ

ตัวแทนเจ้าหนี้เริ่มต้น pacs.003 ไปยังตัวแทนลูกหนี้เพื่อเรียกเก็บเงิน ตัวแทนลูกหนี้ตรวจสอบคำสั่งมอบอำนาจ ตรวจสอบยอดเงินในบัญชี และทำการชำระบัญชีหรือส่งคืนธุรกรรม
