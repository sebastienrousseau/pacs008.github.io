# API

該專案同時提供 REST API 和 CLI，用於營運支付訊息處理工作流程。

## API 功能

- 健康和就緒端點
- XML 產生前的資料驗證
- 直接工作流程的同步產生
- 較長管線的非同步任務執行
- 透過任務完成流程可下載的產生檔案

## CLI 功能

- 指向來源檔案和訊息版本
- 交付前按 XSD 驗證
- 將 XML 產生到管線友善的輸出目錄
- 適配 CI 任務、批次排程和本機操作工具

## 實施重點

Pacs008 專為支付處理團隊的營運使用而設計：

- 訊息建立前的預檢驗證
- 執行時選擇方案和版本
- 內部服務的非同步產生流程
- 用於測試和稽核追蹤的確定性輸出

## 2026 年資料品質要求

訊息品質要求正在全行業收緊，特別是在以下方面：

- 當事方和代理識別
- 結構化或混合地址就緒性
- 更豐富的匯款和參考處理
- 串列支付鏈的透明度

API 和 CLI 旨在將這些檢查作為工作流程的一部分，而非手動審查步驟。
