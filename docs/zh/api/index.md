# API

该项目同时提供 REST API 和 CLI，用于运营支付报文处理工作流。

## API 功能

- 健康和就绪端点
- XML 生成前的数据验证
- 直接工作流的同步生成
- 较长管道的异步任务执行
- 通过任务完成流程可下载的生成文件

## CLI 功能

- 指向源文件和报文版本
- 交付前按 XSD 验证
- 将 XML 生成到管道友好的输出目录
- 适配 CI 任务、批处理计划和本地操作工具

## 实施重点

Pacs008 专为支付处理团队的运营使用而设计：

- 报文创建前的预检验证
- 运行时选择方案和版本
- 内部服务的异步生成流程
- 用于测试和审计跟踪的确定性输出

## 2026 年数据质量要求

报文质量要求正在全行业收紧，特别是在以下方面：

- 当事方和代理识别
- 结构化或混合地址就绪性
- 更丰富的汇款和参考处理
- 串行支付链的透明度

API 和 CLI 旨在将这些检查作为工作流的一部分，而非手动审查步骤。
